## 2. 项目目录结构（已创建）

```text
project_root/
  data/
    raw/               # 原始数据（下载/解压后）
    processed/         # 抽帧/resize/缓存后的数据
    splits/            # few-shot索引、val索引、（可选）任务抽样索引
  configs/
    dataset/           # 数据集与路径、抽帧策略等
    model/             # 模型名/权重/冻结策略等
    train/             # 训练超参（lr/epoch/aug/amp等）
    eval/              # 评估协议（Protocol A/B 参数等）
  src/                 # 代码主体（dataset/model/utils/metrics等）
  tools/               # CLI入口脚本（prepare/train/eval/benchmark/summarize）
  runs/                # 单次运行产物（按时间戳/实验ID组织）
  results/
    tables/            # 汇总表（csv）
    figures/           # 图（png/pdf）
    summaries/         # 汇总json/markdown（可选）
  docs/                # 长期文档（本文档等）
```

## 3. 运行入口（tools/）规划清单（先写“最小可跑版本”，再迭代）

> 约定：每次运行必须把实际使用的配置拷贝到 `runs/<run_id>/config/`，并输出 `runs/<run_id>/metrics.json`。

- **`tools/prepare_dataset.py`**
  - 目的：下载/解压/校验；（可选）抽帧到 `data/processed/`；生成 `label2id.json`、类别统计。
  - 输入：数据源路径或下载链接、抽帧参数（T_train/T_test）、输出目录。
  - 输出：`data/raw/`、`data/processed/`、基础统计（每类样本数）。

- **`tools/make_fewshot_splits.py`**
  - 目的：生成 K-shot 训练索引与 val 索引（若官方无 val）；保存到 `data/splits/`。
  - 输入：官方划分文件、K、seed、每类 val 数 V。
  - 输出：`data/splits/kshot_K{K}_seed{seed}.json`、`data/splits/val_seed{seed}.json` 等。

- **`tools/train.py`**
  - 目的：按配置训练（两阶段：训头/全网），保存 best checkpoint。
  - 输入：dataset/model/train config；seed；输出 run_id。
  - 输出：`runs/<run_id>/checkpoints/best.pt`、训练日志（可选 TensorBoard）。

- **`tools/eval.py`**
  - 目的：统一评估 Protocol A/B；输出 Top-1/Top-5、稳定性、class-wise 等。
  - 输入：checkpoint、eval config、splits。
  - 输出：`runs/<run_id>/metrics.json`、（可选）confusion matrix、per-class acc。

- **`tools/benchmark.py`**
  - 目的：输出 Params/FLOPs/Latency/Throughput/Peak VRAM（按计划中的测量流程）。
  - 输入：模型配置、输入尺寸、batch（1/16）、AMP 开关。
  - 输出：`runs/<run_id>/benchmark.json`。

- **`tools/summarize_results.py`**
  - 目的：聚合多 seed、多模型、多 K 的结果，生成最终表/图。
  - 输入：runs 目录或结果清单。
  - 输出：`results/tables/*.csv`、`results/figures/*.png`。
