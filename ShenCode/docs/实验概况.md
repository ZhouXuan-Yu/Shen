# 实验概况

本文档详细介绍实验的视频/图片处理方式、模型列表、K值设置、样本位置等核心信息。

---

## 一、视频与图片处理方式

### 1.1 数据类型

本项目使用的是 **视频数据**，而非静态图片。

| 属性 | 说明 |
|------|------|
| **原始格式** | MP4 视频文件 |
| **视频类型** | RGB 彩色视频（`*_color.mp4`）和深度视频（`*_depth.mp4`） |
| **使用类型** | 仅使用 RGB 彩色视频 |
| **文件命名** | `signer{X}_sample{Y}_color.mp4` |

### 1.2 视频处理流程

```
原始视频 (MP4)
     │
     ▼
┌─────────────────────────────────────────────────────────────┐
│ 步骤 1: 视频加载                                             │
│   - 使用 decord 库（首选）或 OpenCV 读取视频                 │
│   - 获取视频总帧数                                           │
└─────────────────────────────────────────────────────────────┘
     │
     ▼
┌─────────────────────────────────────────────────────────────┐
│ 步骤 2: 帧采样                                               │
│   - 从视频中采样 8 帧（num_frames=8）                        │
│   - 采样策略：uniform（均匀采样）                            │
│   - 例：100帧视频 → 采样第 0, 12, 25, 37, 50, 62, 75, 87 帧  │
└─────────────────────────────────────────────────────────────┘
     │
     ▼
┌─────────────────────────────────────────────────────────────┐
│ 步骤 3: 图像变换                                             │
│   训练时：                                                   │
│   - Resize: 调整为 224×224 像素                              │
│   - RandomHorizontalFlip: 随机水平翻转（p=0.5）              │
│   - ColorJitter: 颜色抖动（亮度、对比度、饱和度各 ±20%）     │
│   - Normalize: ImageNet 标准化                               │
│                                                              │
│   评估时：                                                   │
│   - Resize: 调整为 224×224 像素                              │
│   - Normalize: ImageNet 标准化                               │
└─────────────────────────────────────────────────────────────┘
     │
     ▼
┌─────────────────────────────────────────────────────────────┐
│ 步骤 4: 帧聚合                                               │
│   - 8 帧分别通过 CNN 提取特征                                │
│   - 对 8 帧的输出取平均（mean pooling）                      │
│   - 得到视频级别的预测结果                                   │
└─────────────────────────────────────────────────────────────┘
     │
     ▼
最终输出：226 类的概率分布
```

### 1.3 帧采样策略

| 策略 | 说明 | 使用场景 |
|------|------|----------|
| **uniform** | 均匀采样，帧间隔相等 | 默认策略，训练和评估 |
| random | 随机采样指定数量的帧 | 可选，增加多样性 |
| center | 从视频中间采样连续帧 | 可选，聚焦核心动作 |

### 1.4 图像标准化参数

使用 ImageNet 预训练模型的标准化参数：

```python
mean = (0.485, 0.456, 0.406)  # RGB 通道均值
std  = (0.229, 0.224, 0.225)  # RGB 通道标准差
```

## 二、完整模型列表

### 2.1 模型总览（共 8 个）

| 编号 | 模型名称 | 代码标识 | 参数量 | 特点 | 来源 |
|------|----------|----------|--------|------|------|
| 1 | ResNet-18 | `resnet18` | 11.29M | 经典残差网络，18层 | torchvision |
| 2 | ResNet-34 | `resnet34` | 21.40M | 更深的残差网络，34层 | torchvision |
| 3 | ResNet-50 | `resnet50` | 25.56M | 瓶颈结构，50层 | torchvision |
| 4 | MobileNetV2 | `mobilenetv2` | 2.51M | 深度可分离卷积，倒残差 | torchvision |
| 5 | MobileNetV3-Small | `mobilenetv3_small` | 1.52M | NAS搜索，最轻量 | torchvision |
| 6 | MobileNetV3-Large | `mobilenetv3_large` | 4.21M | NAS搜索，性能更优 | torchvision |
| 7 | ShuffleNetV2 | `shufflenetv2` | 2.28M | 通道混洗，高效轻量 | torchvision |
| 8 | EfficientNet-B0 | `efficientnet_b0` | 5.29M | 复合缩放，效率优化 | torchvision |

### 2.2 模型分组

| 分组 | 模型 | 研究目的 |
|------|------|----------|
| **ResNet 系列** | ResNet-18, 34, 50 | 研究网络深度对小样本学习的影响 |
| **MobileNet 系列** | MobileNetV2, V3-S, V3-L | 研究轻量化架构在小样本场景的表现 |
| **其他轻量模型** | ShuffleNetV2, EfficientNet-B0 | 对比不同轻量化策略 |

## 三、K 值设置（共 6 个）

### 3.1 K 值含义

**K-shot Learning**：每个类别仅使用 K 个样本进行训练。

### 3.2 完整 K 值列表

| K 值 | 含义 | 每类训练样本 | 总训练样本（226类） | 占原训练集比例 |
|------|------|--------------|---------------------|----------------|
| K=1 | 极端小样本 | 1 个 | 226 个 | 0.80% |
| K=2 | 极少样本 | 2 个 | 452 个 | 1.61% |
| K=5 | 标准小样本 | 5 个 | 1,130 个 | 4.01% |
| K=10 | 中等小样本 | 10 个 | 2,260 个 | 8.03% |
| K=20 | 较多样本 | 20 个 | 4,520 个 | 16.05% |
| K=50 | 丰富样本 | 50 个 | 11,300 个 | 40.14% |

## 四、样本位置与文件类型

```
data/
├── raw/                          # 原始数据
│   ├── train/
│   │   ├── train/                # 训练视频
│   │   │   ├── signer0_sample1000_color.mp4
│   │   │   ├── signer0_sample1000_depth.mp4
│   │   │   └── ...
│   │   └── train_labels.csv      # 训练标签
│   ├── val/
│   │   ├── val/                  # 验证视频
│   │   └── val_labels.csv        # 验证标签
│   └── test/
│       ├── test/                 # 测试视频
│       └── test_labels.csv       # 测试标签
│
├── processed/                    # 预处理后的数据
│   ├── manifest.jsonl            # 样本清单（路径、标签、划分）
│   ├── id2label.json             # ID 到标签名的映射
│   ├── label2id.json             # 标签名到 ID 的映射
│   └── stats.json                # 数据集统计信息
│
└── splits/                       # K-shot 划分文件
    ├── kshot_K1_seed0.json       # K=1, seed=0 的训练样本索引
    ├── kshot_K1_seed1.json
    ├── kshot_K1_seed2.json
    ├── kshot_K5_seed0.json       # K=5, seed=0 的训练样本索引
    ├── ...
    └── val_seed*.json            # 验证集索引
```

### 4.2 文件类型说明

| 文件类型 | 扩展名 | 位置 | 用途 |
|----------|--------|------|------|
| **视频文件** | `.mp4` | `data/raw/{split}/{split}/` | 原始手语视频 |
| **标签文件** | `.csv` | `data/raw/{split}/` | 视频与标签的对应关系 |
| **样本清单** | `.jsonl` | `data/processed/` | 所有样本的路径、标签、划分信息 |
| **划分索引** | `.json` | `data/splits/` | K-shot 实验的训练样本索引 |
| **映射文件** | `.json` | `data/processed/` | 标签名与数字 ID 的映射 |

### 4.3 manifest.jsonl 格式

每行一个 JSON 对象：

```json
{
  "global_id": 0,
  "path": "data/raw/train/train/signer0_sample1000_color.mp4",
  "label": "greeting",
  "label_id": 42,
  "split": "train",
  "signer": 0
}
```

### 4.4 训练时使用的文件

| 阶段 | 使用文件 | 说明 |
|------|----------|------|
| **加载样本清单** | `data/processed/manifest.jsonl` | 获取所有样本信息 |
| **加载 K-shot 划分** | `data/splits/kshot_K{k}_seed{s}.json` | 获取训练样本索引 |
| **读取视频** | `data/raw/train/train/*.mp4` | 加载 RGB 彩色视频 |
| **标签映射** | `data/processed/label2id.json` | 标签名转数字 ID |

## 五、单次训练详细流程

以 `ResNet-18, K=5, seed=0` 为例：

### 5.1 数据加载

1. 读取 `data/splits/kshot_K5_seed0.json`，获取 1,130 个训练样本索引
2. 读取 `data/processed/manifest.jsonl`，根据索引筛选样本
3. 每个样本对应一个 `.mp4` 视频文件

### 5.2 训练过程

```
输入：1,130 个视频样本（226类 × 5样本/类）
       ↓
视频读取：每个视频采样 8 帧
       ↓
数据增强：Resize + RandomFlip + ColorJitter + Normalize
       ↓
模型前向：8 帧分别通过 ResNet-18，输出取平均
       ↓
损失计算：CrossEntropy + LabelSmoothing(0.1)
       ↓
反向传播：Adam 优化器更新参数
       ↓
重复 50 epochs
```

### 5.3 输出文件

```
runs/resnet18_K5_seed0_20260105_112233/
├── checkpoints/
│   ├── best.pt          # 最佳模型权重
│   ├── last.pt          # 最后一轮权重
│   └── epoch_*.pt       # 定期保存的权重
├── metrics.json          # 训练曲线数据
├── results.json          # 最终结果摘要
└── config.yaml           # 训练配置
```
