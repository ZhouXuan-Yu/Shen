# 《简化版 ResNet 在小样本手语识别中的应用》

开发计划 + 实验计划（工程可执行版，3 天内完成）

## 0. 项目总览

- **项目定位**
  - 本题重点是“应用与对比分析”，不是算法创新。
  - 目标：将多种**现成**简化 ResNet（至少 ResNet-18/34）应用于小样本手语识别设置，并与 MobileNetV2 等轻量模型做横向对比，形成模型选型建议。

- **教师硬约束（必须遵守）**
  - 禁止给任何“改模型结构/设计新模型”的建议与实现。
  - 只能：
    - 选哪些现成模型（torchvision / timm / 公开实现）
    - 在统一规则下做公平对比
    - 组织分析与论文证据链
  - 允许做的只有：替换最后分类层以适配类别数；训练策略层面允许调参。

- **资源约束（用于裁剪计划）**
  - GPU：RTX 3050 4GB（移动版）
  - 总实验时间控制在几天内（本计划给出 3 天版本）
  - 显存有限：需控制 batch size、视频帧数、缓存策略、实验优先级

## B. 模型选择方案（只限现成模型）

### B1. 候选模型清单（含理由与获取途径）

- **P0（必须跑）**
  - **ResNet-18**（torchvision/timm）
    - 理由：经典基线、容量适中、小样本下易复现。
  - **ResNet-34**（torchvision/timm）
    - 理由：更高容量，便于分析“深度/容量 vs 过拟合”的关系。
  - **MobileNetV2**（torchvision）
    - 理由：移动端/轻量代表，资源约束下的强对照。

- **P1（建议扩展，增强论文说服力）**
  - **ShuffleNetV2 1.0x**（torchvision）
    - 理由：极轻量、低延迟，适合作为“速度上限”对照。
  - **EfficientNet-B0**（torchvision/timm）
    - 理由：轻量且精度强，适合作为“性价比更优”对照。

### B2. 预训练策略（公平性处理）

- **P0 主线**：所有模型统一使用 **ImageNet 预训练权重**。
- **P1 变量**：在 1–2 个代表模型上做 **Pretrain vs Scratch**（如 ResNet18、MobileNetV2），用于论文中讨论“预训练对小样本的重要性”。

## C. 统一对比实验设计（公平性原则写清）

### C0. 公平性原则（论文可直接复述）

- 同数据集、同官方划分、同 few-shot 采样索引（保存到文件，版本化）。
- 同输入分辨率、同抽帧规则、同归一化（若使用 ImageNet 预训练，则统一采用 ImageNet mean/std）。
- 同训练预算（epoch 上限、early stop、AMP 开关等）与同评估协议（Protocol A/B 参数、seeds 次数）。
- 允许针对不同模型微调超参，但必须：
  - 仅在给定范围内搜索
  - 仅在验证集上决定
  - 完整记录搜索过程与最终选择

### C1. 数据集选择（推荐 + 备选）

- **主推荐（P0）**：优先选择有公开版本且常用的手语识别基准数据集，并使用作者/官方提供的划分。
  - 可优先考虑 **WLASL100/WLASL200**（若下载链路可用，且有官方划分）。
- **备选（P1）**：若主推荐数据出现不可稳定下载/缺失（例如视频源失效），立即切换到同样具备固定划分、可复现下载的数据集（以“可复现获取”为第一优先级）。

- **划分与验证集策略（保证可复现）**
  - 优先使用数据集官方提供的 `train/val/test`（或等价）划分。
  - 若官方仅提供 `train/test`：从 `train` 中按类固定抽取 `V=2` 个样本作为 `val`（使用固定 seed 生成索引并保存），其余为 few-shot 训练池。

- **数据集定稿检查清单（S0 必须通过）**
  - [ ] 可稳定下载/可复现获取（下载链接或脚本可用；必要时记录校验信息）
  - [ ] 许可、引用信息明确（论文可写清数据来源与引用）
  - [ ] 官方划分可解析：`train/val/test` 或 `train/test` 文件齐全、字段含义明确
  - [ ] 划分无泄漏：同一视频/同一实例不跨 split（至少按唯一 ID 去重检查）
  - [ ] 类别集合定稿：最终类别数 C、类别名/ID 映射表固定（`label2id`/`id2label`）
  - [ ] 样本审计：输出每类样本数统计；满足 `train` 每类 ≥ `K_max`；`test` 每类 ≥ `Q`（Protocol B）或能覆盖 Protocol A 全量测试
  - [ ] 异常样本可控：坏视频/坏图片比例可接受（例如 <1%）；剔除规则固定并记录
  - [ ] 输入形式可落地：视频可抽帧；抽帧策略、默认输入尺寸、归一化策略已确定
  - [ ] 存储与 I/O 预算可承受：抽帧后数据量/磁盘占用评估；训练读取速度可接受
  - [ ] 复现实验所需索引文件落盘：val 索引、K-shot 索引、（可选）Protocol B 任务抽样索引

> 执行要求：Day1 上午完成“可下载性验证 + 类别/样本数审计”，半天内确定最终数据集，避免后续被数据卡住。

### C2. 输入形式与预处理（不改结构）

- 由于 backbone 为 2D CNN（ResNet/MobileNet 等），主线采用 **视频抽帧 → 图像输入**。
- 文本/标签（如 gloss/类别名）仅作为监督信号与统计分析信息，不作为模型输入模态，避免引入多模态结构与结构改造争议。
- **抽帧默认值（4GB 友好）**
  - MVP：`T_train=1`（每视频 1 帧），`T_test=3`
  - 扩展：`T_train=3`，`T_test=5`
- **视频级推理（可选，不改结构）**：对同一视频抽取 T 帧分别前向，对 logits 取平均得到视频预测。
- **输入尺寸**：默认 `224×224`；如显存紧张，对所有模型统一降到 `192` 或 `160` 并记录。

### C3. 小样本设置（K-shot 训练 + Protocol A/B 评估）

- **训练集 K-shot（必须固化索引）**：每类仅抽取 K 个样本参与训练（few-shot 索引文件必须保存）。

- **Protocol A：固定划分的全量测试（默认主协议，贴近真实落地）**
  - 数据划分：严格使用官方 `train/val/test`（或等价）；若无 `val` 则按前述策略从 `train` 固定抽取。
  - 每个 seed：在 `train` 上按类抽 K 个样本训练；在 `val` 上选最佳；在 `test` 上对全部测试样本做一次性评估。
  - 统计方式：跨 seeds 报告 `mean±std`；并保留每次 seed 的单次结果用于稳定性分析。

- **Protocol B：按任务抽样的子集评估（用于论文增强说服力，仅评估侧做任务抽样统计）**
  - 目的：在固定模型与固定 `test` split 上，通过“抽取若干个 N 类子任务”来观察模型在不同类别组合下的波动。
  - 默认参数：
    - N-way：`N=5`（主）+ `N=10`（补充）
    - 每类测试样本数：`Q=15`（不足则统一降到 `Q=5` 并记录）
    - 任务数：`T=200`/seed（MVP），关键配置可提升到 `T=600`
  - 计算方式（不改结构，仅评估侧处理）：每个任务从 `test` 中选 N 个类别并抽取对应样本，统计该任务准确率；最终对任务维度求均值，再对 seeds 求 `mean±std`。

- **Seeds（重复次数）**：P0 用 `3` 次（0/1/2），最终主结论可扩展到 `5` 次（0–4）

### C4. 统一训练策略（默认值 + 可选范围）

- **Loss（默认且建议统一）**：Cross-Entropy（避免引入 few-shot 特有损失导致“算法创新”争议）。
- **分类头**：仅替换最后分类层以适配类别数，其余结构不变。
- **冻结策略（两段式，默认）**
  - 阶段A：仅训练分类头 5 epoch
  - 阶段B：解冻全网络训练至早停（上限 50 epoch）
- **优化器**：AdamW
  - 默认：训头 `lr=1e-3`，全网 `lr=3e-4`
  - 范围：`[1e-4, 3e-3]`
  - `weight_decay=1e-4`（范围 `[0, 5e-4]`）
- **学习率策略**：Cosine decay，warmup 3 epoch（范围 0–5）
- **Batch size**：默认 32（范围 8/16/32/64；OOM 则对所有模型统一下调并记录）
- **AMP**：开启（显存友好）
- **Early stop**：patience 10（监控 val acc；如无 val，则固定 epoch 并重点报告波动/过拟合）

- **默认增强清单（训练端，P0 建议保持一致）**
  - Resize/CenterCrop（或 RandomResizedCrop，作为默认强度）
  - 轻量颜色扰动：ColorJitter（亮度/对比度/饱和度/色相，弱强度）
  - 随机灰度（弱强度）
  - 轻量几何扰动：小角度旋转/平移（弱强度）
  - RandomErasing（弱强度，可选开关，但需对所有模型一致）
  - 说明：水平翻转可能改变手势语义，默认关闭；如启用必须在论文中说明并对所有模型一致。

- **调参预算化规则（保证 3 天内能跑完）**
  - 调参只在验证集上决定；最终 `test` 只用于一次性报告（避免“看 test 调参”）。
  - P0 只允许调以下 3 类“低风险”超参：学习率、weight decay、增强强度（不引入新损失/新结构）。
  - 预算建议（每个模型，每个 K）：
    - 1 次默认配置跑通（必做）
    - 2–4 次学习率/weight decay 小网格（只用 `seed=0`，且可用更小 `T`/更少 epoch 做快速筛选）
    - 1 次确认跑（用选定配置 + `seed=0` 跑到完整预算）
  - 迁移原则：先在 ResNet-18（K=5）上确定“全局默认配置”，其余模型仅允许做 0–2 次学习率缩放确认（例如 0.5×/1×/2×）。
  - 全过程必须记录：每次 trial 的 config、验证集指标与选择理由（保存在 `runs/` 与 `results/`）。

### C5. 评估指标与资源指标

- **精度指标**：Top-1（必须）、Top-5（推荐）、mean±std（跨 seeds；Protocol B 还包含任务维度统计）
- **训练行为**：收敛速度（最佳 val 所在 epoch）、过拟合程度（train-val gap）
- **稳定性指标（必须补齐）**
  - seeds 维度：Std、95% CI（或 Std/均值 的相对波动）、最差 seed 与最好 seed 差值
  - Protocol B 任务维度：任务内准确率的 Std（反映“类别组合敏感性”）
- **类别维度指标（必须补齐）**
  - 每类准确率（class-wise acc），并报告：Macro 平均、最差/最好 Top-K 类别、25/50/75 分位数
  - 混淆矩阵（挑选 P0 模型 + 关键 K 输出），用于论文中解释易混淆类别
  - 类别不均衡关联：样本数 vs 类别准确率散点（辅助分析长尾问题）
- **资源/效率指标**：Params、FLOPs、Latency、Throughput、Peak VRAM

### C6. Params/FLOPs/Latency 的测量环境与流程（可复现）

- 记录：GPU 型号、torch/cuda/driver、输入尺寸、batch、AMP 开关。
- Latency：batch=1 与 batch=16 各测一次；warmup 50；计时 200 次；报告均值与 P95。

- Throughput（吞吐量，补充）
  - 建议同时给出：batch=16 的 inference 吞吐（samples/s）与训练端吞吐（可选，images/s 或 frames/s）。
  - 计时规则：与 Latency 同环境；固定输入张量；warmup 后计时多次取均值。

- Peak VRAM（显存峰值，补充）
  - 训练端：记录一次典型训练 step 的 `max_memory_allocated`（或等价峰值指标），单位 MB。
  - 推理端：记录 batch=1 与 batch=16 的峰值显存（便于论文资源对比）。

## G. 风险清单与备选方案（只能工程/实验层面）

- 数据不足/类别不均衡：定义可用类别集合规则；固化筛选名单与 few-shot 索引。
- 小样本波动大：多 seed（≥3），报告 mean±std。
- 训练不稳定/过拟合：early stop、冻结策略更保守、降低 lr、调整增强（不改结构）。
- 算力不足/时间不足：优先完成 P0；减少帧数/Protocol B 任务数 T；P1 按优先级逐步加。

## I. 3 天内推荐执行顺序（按半天粒度）

- **Day1 上午**：数据集可下载性验证 + 样本审计 + 定稿（输入形式/抽帧/尺寸）
- **Day1 下午**：跑通单实验（ResNet18 K=5）+ 固化 splits（K=1/5，seed=0/1/2）
- **Day2 上午**：ResNet34 批量跑（K=1/5，3 seeds）
- **Day2 下午**：MobileNetV2 批量跑 + 补齐 Params/FLOPs/Latency/Throughput/Peak VRAM + 生成表1/图3
- **Day3 上午**：P1 扩展（加 1–2 个轻量模型或做 pretrain vs scratch）
- **Day3 下午**：生成收敛/过拟合图 + 汇总最终表 + 输出模型选型建议文本
