模型训练一轮完整跑通操作流程（CE-CSL，2 阶段：特征提取 + CTC 训练）
================================================

一、环境与数据准备
----------------
1. 确保已经安装好 Python/Anaconda 和依赖（torch、torchvision、opencv-python、pandas 等）。
2. 确认数据集目录结构（默认）：
   - 数据根目录：`D:\Aprogress\Shen\dataset\CE-CSL\CE-CSL`
   - 其中应包含：
     - `video\train`、`video\dev`、`video\test`（或对应子目录 A~L 的 MP4 视频）
     - `label\train.csv`、`label\dev.csv`、`label\test.csv`
3. 打开命令行（Anaconda Prompt 或 PowerShell），进入代码目录：

   ```bash
   cd D:\Aprogress\Shen\AAA
   ```

二、阶段一：提取视频帧特征（ResNet18，变长特征）
------------------------------------------
1. 执行以下命令：

   ```bash
   python extract_features.py --splits train,dev,test --max-samples 0
   ```

2. 这条命令的含义：
   - `extract_features.py`：对 CE-CSL 的视频逐帧跑 ResNet18，提取 512 维特征。
   - `--splits train,dev,test`：对 `train`、`dev`、`test` 三个数据划分都进行特征提取。
   - `--max-samples 0`：0 表示不限制样本数量，即把该划分下的所有视频都处理一遍（完整数据）。
3. 运行完成后，会在数据根目录下生成特征文件目录：
   - `D:\Aprogress\Shen\dataset\CE-CSL\CE-CSL\train_features`
   - `D:\Aprogress\Shen\dataset\CE-CSL\CE-CSL\val_features`
   - `D:\Aprogress\Shen\dataset\CE-CSL\CE-CSL\test_features`
   每个视频对应一个 `.npy` 文件，形状为 `(T, 512)`（T 是该视频的帧数，保留变长）。

> 提示：如果只是想先快速测试流程，可以临时改成：
>
> ```bash
> python extract_features.py --splits train,dev --max-samples 10
> ```
>
> 只对 train/dev 各抽 10 个样本提特征，会快很多，确认代码能跑通后再跑完整数据。

三、阶段二：基于特征的 1D-CNN + BiLSTM + CTC 训练（跑 1 个 epoch）
-----------------------------------------------------------
1. 在同一目录下，执行训练脚本：

   ```bash
   python train_lstm.py
   ```

2. 该脚本会自动完成以下工作：
   - 从 `label\train.csv` 和 `label\dev.csv` 中读取 `Gloss` 字段并构建词表（包含 `<blank>`）。
   - 从 `train_features`、`val_features` 中读取 `.npy` 特征文件，与标签按 `Number_Translator` 对应。
   - 使用 `1D-CNN + BiLSTM + CTC` 的模型结构：
     - 先用 `Conv1d + MaxPool1d` 对时间维做下采样；
     - 再用双向 LSTM 进行时序建模；
     - 最后通过全连接层输出到各个 gloss 类别，然后用 CTC Loss 训练。
   - 使用动态 padding + 排序 + `pack_padded_sequence` 处理变长序列。
   - 训练 **1 个 epoch**（当前配置已设置为 1），同时在 dev 集上做验证，打印 `Train CTC Loss` 和 `Val CTC Loss`。
   - 根据 dev 集 CTC Loss 自动保存效果最好的模型到：
     - `output\ctc_lstm\best_model.pt`

3. 当命令执行结束、控制台打印出 `Epoch 1/1` 的训练与验证损失，并提示保存了 `best_model.pt`，说明：
   - 整个“特征提取 + CTC 训练 1 个 epoch”流程已经完整跑通。

----------------
1. 如果训练脚本一开始就报“找不到特征目录/特征文件”，优先检查：
   - 是否已经成功跑完 `extract_features.py`，且生成了 `train_features/val_features/test_features`。
   - `DATA_ROOT` 配置是否与实际数据路径一致。
2. 如果想再次训练，只需保证特征已经存在，直接重复执行：

   ```bash
   cd D:\Aprogress\Shen\AAA
   python train_lstm.py
   ```

   不需要重新提取特征，除非你修改了特征提取策略或换了数据。
