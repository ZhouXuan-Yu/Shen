# HandTalk AI 项目技术分析报告

## 一、项目概述

**HandTalk AI（译手）** 是一个基于深度学习的手语识别与翻译系统，旨在帮助聋哑人群与健听人群进行无障碍沟通。项目采用前后端分离架构，集成了多种深度学习模型，实现了从手语视频/图片到中文文本的端到端识别与翻译。

### 核心功能模块

1. **手语词典搜索**：支持关键词、语义、混合搜索的手语词汇查询系统
2. **手语识别**：基于 CTC 模型的连续手语识别（视频/图片）
3. **文本→视频检索**：基于向量检索的 CE-CSL 数据集视频推荐
4. **实时识别**：基于摄像头的实时手语识别（HTTP 轮询/WebSocket）

# 三种搜索模式

### 3.2 CTC 连续手语识别系统

#### 功能描述
基于 CTC（Connectionist Temporal Classification）损失函数的端到端连续手语识别，支持视频和图片输入。

#### 模型架构

**三层架构**（`AAA/ctc_model.py`）：

```
输入视频帧序列 (B, T, H, W, C)
    ↓
[1] ResNet18 特征提取器
    ↓ 提取每帧的 512 维特征
特征序列 (B, T, 512)
    ↓
[2] 1D 时序卷积层（可选，kernel_size=3, padding=1, stride=1）
    ↓ 局部时序建模，不改变时间长度
时序特征 (B, T, 512)
    ↓
[3] 双向 LSTM（2 层，hidden_size=256，bidirectional=True）
    ↓ 全局时序建模
LSTM 输出 (B, T, 512)  # 256*2=512
    ↓
[4] 全连接分类层
    ↓ 输出到词汇表大小
Logits (T, B, num_classes)
    ↓
[5] CTC Loss + Greedy Decode
    ↓
输出：Gloss 序列（手语词汇序列）
```

**关键技术点**：

1. **特征提取**（`AAA/extract_features.py`）：
   - 使用预训练 ResNet18 提取每帧的 512 维特征
   - 支持视频帧采样（默认 32 帧/视频）
   - 图像预处理：Resize(224×224), Normalize(ImageNet 均值/方差)

2. **时序建模**：
   - **1D-CNN**：在时间维度上做卷积，kernel_size=3, padding=1, stride=1，**保证时间长度 T 不变**（避免 CTC 报错 T < L）
   - **BiLSTM**：双向 LSTM 捕获前后文信息，使用 `pack_padded_sequence` 提升变长序列计算效率

3. **CTC 解码**（`AAA/demo_infer.py`）：
   ```python
   # Greedy 解码流程
   1. 每个时间步取概率最大的类别
   2. 折叠重复的 ID（连续相同 ID 合并）
   3. 去掉 blank（ID=0）
   4. 将 ID 映射回字符/词汇
   ```

4. **词汇表构建**（`AAA/ctc_dataset.py`）：
   - 从训练数据中统计所有字符/词汇
   - 特殊 token：`<blank>` (ID=0), `<unk>` (ID=1)
   - 支持字符级和词汇级两种模式

**训练流程**（`AAA/train_lstm.py`）：
- **数据集**：CE-CSL（Chinese Sign Language）数据集
- **损失函数**：CTC Loss（`torch.nn.CTCLoss`）
- **优化器**：Adam，学习率 1e-4
- **训练策略**：梯度裁剪（max_grad_norm=1.0），早停机制

**推理接口**（`app/ctc_service.py`）：
- `recognize_video_file(video_path)`：处理视频文件
- `recognize_image_file(image_path)`：处理图片（视为单帧视频）

**模型文件**：
- 默认路径：`Project/Back/model/last_checkpoint.pt`
- 包含：模型权重、词汇表（Vocabulary）、配置信息

### 3.3 文本→视频检索系统（Video RAG）

#### 功能描述
基于向量检索的文本到视频推荐系统，用户输入自然语言句子，系统返回最相似的手语视频。

#### 技术实现

**向量模型**：
- 使用 `sentence-transformers` 库
- 模型：`BAAI/bge-small-zh-v1.5`（中文句向量模型）
- 向量维度：384 维

**索引构建**（`scripts/build_cecsl_index.py`）：
1. 读取 CE-CSL 数据集的标注文件（dev.csv）
2. 对每条句子进行清洗（去空白、统一标点）
3. 使用句向量模型编码为 384 维向量
4. 保存为：
   - `embeddings_{split}.npy`：所有向量的矩阵 (N, 384)
   - `metadata_{split}.json`：每条记录的元数据（sample_id, sentence, video_path 等）

**检索算法**（`app/rag_index.py`）：
```python
1. 将查询文本编码为向量（query_vec）
2. 计算与所有索引向量的余弦相似度（归一化点积）
3. 使用 np.argpartition 快速获取 top_k
4. 返回结果：sentence, videoUrl, similarity, rank
```

**API 接口**：
- `GET /api/v1/video_rag/search?query=句子&top_k=10`
- `GET /api/v1/video_rag/random?limit=10`

**视频服务**：
- 本地视频目录挂载：`/cecsl/video` → `dataset/CE-CSL/CE-CSL/video`
- 用户上传视频：`/uploads/video` → `uploads/video`
- 通过 FastAPI `StaticFiles` 提供静态文件服务

## 七、关键技术细节

### 7.1 CTC Loss 原理

**问题**：手语视频帧数（T）与输出词汇数（L）不对齐，无法直接使用交叉熵损失。

**解决方案**：CTC Loss 允许：
- 输入序列长度 T 可以大于输出序列长度 L
- 通过引入 `blank` token 处理对齐问题
- 使用动态规划（前向-后向算法）计算所有可能对齐路径的概率

**解码方式**：
- **Greedy Decode**：每个时间步取概率最大的类别，然后折叠重复、去掉 blank
- **Beam Search**（可选）：保留 top-k 路径，更准确但更慢

### 7.2 向量检索优化

**当前实现**：纯 NumPy + 余弦相似度
- 优点：简单、无需额外依赖
- 缺点：O(N) 时间复杂度，N 为索引大小

**优化方向**（未来）：
- 使用 FAISS（Facebook AI Similarity Search）构建 AnnIndex
- 支持近似最近邻搜索（ANN），时间复杂度降至 O(log N)
- 支持 GPU 加速

### 7.3 前端状态管理

**Pinia Stores**：
- `stores/auth.ts`：用户认证状态
- `stores/dictionary.ts`：词典搜索状态
- `stores/recognition.ts`：识别结果状态
- `stores/videoRag.ts`：视频检索状态

**数据流**：
```

```
